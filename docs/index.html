🧠 The Intelligence Stack (IQ Layers)

The IQ Stack represents the intelligence outcomes Microsoft delivers. These are not separate SKUs — they are architectural layers that compound in value when used together.

1

Work IQ — Collaboration Intelligence

Purpose: Understands how people work and collaborate

Sources: Teams, Outlook, M365 Documents, Graph API, Calendar, Viva Insights

Use Cases: Meeting insights, email triage, document co-authoring, workflow triggers, People analytics

2

Fabric IQ — Business State Intelligence

Purpose: Represents the live state and actions of the business

Sources: OneLake, Microsoft Fabric, Power BI, SQL, Real-time telemetry, Dynamics 365

Use Cases: Real-time BI, operational dashboards, data agents, predictive analytics, data pipelines

3

Foundry IQ — Institutional Knowledge

Purpose: Provides curated institutional knowledge via Agentic Retrieval (RAG)

Sources: SharePoint, Web, Blob Storage, MCP Servers, External APIs

Use Cases: Knowledge bases, policy retrieval, documentation search, best-practice agents, compliance guides

🔌 Model Context Protocol (MCP)

MCP standardizes how agents interact with external data sources, creating a universal connector layer.

Supported Integrations

  • Dynamics 365 (Finance & Operations, CRM)
  • Workday (HR, Payroll)
  • Oracle (ERP, DB)
  • SAP
  • Custom REST / GraphQL APIs
Best Practice: Build MCP servers as thin adapters — don't embed business logic. Keep them stateless and versioned.

💻 MCP Server Example

// Example: MCP server for Dynamics 365 F&O export const dynamics365Tool = { name: "get_vendor_invoices", description: "Retrieve open vendor invoices", parameters: { vendorId: { type: "string", required: true }, status: { type: "string", enum: ["Open","Posted"] } }, handler: async ({ vendorId, status }) => { // Respects user-level D365 permissions return await d365Client.invoices.list({ vendorId, status }); } };

🔑 Graph API Permissions

Fine-grained Graph API permissions control exactly what each agent can access — no over-privileging.

🏷️ Confidentiality Inheritance

Compliance Data

Purview sensitivity labels are automatically respected by Copilot and all agents — no code needed.

  • Agents inherit the permissions of the triggering user
  • Labels control data access down to individual message level
  • Full audit trail for every AI interaction
Best Practice: Label all data in Purview before enabling AI agents. Test with each sensitivity tier.

  • Verify Explicitly: Always authenticate and authorize — every request, every agent
