🧠 The Intelligence Stack (IQ Layers)

The "IQ Stack" represents the intelligence outcomes Microsoft wants organizations to achieve. These are not separate SKUs but architectural layers that work together.

1

Work IQ

Purpose: Understands how people work and collaborate

Data Sources: Teams, Outlook, M365 Documents, Graph API

Use Cases: Meeting insights, email prioritization, document collaboration, workflow automation

2

Fabric IQ

Purpose: Represents the live state and actions of the business

Data Sources: OneLake, Fabric, Power BI, SQL, Real-time telemetry

Use Cases: Business intelligence, real-time analytics, operational dashboards, data warehousing

3

Foundry IQ

Purpose: Provides curated institutional knowledge

Data Sources: SharePoint, Web, Blob Storage, MCP Servers

Use Cases: RAG (Retrieval-Augmented Generation), knowledge bases, documentation, best practices

🔄 How IQ Layers Work Together

The three IQ layers create a comprehensive intelligence ecosystem:

  • Work IQ captures human collaboration patterns
  • Fabric IQ provides real-time business state
  • Foundry IQ offers institutional knowledge
Implementation Strategy: Start by mapping your existing data sources to each IQ layer. Ensure proper data governance and access controls are in place before enabling agentic workflows.

🏷️ Confidentiality Inheritance

Compliance Data

Sensitivity labels from Purview are automatically respected by Copilot and Agents.

How It Works

  • Agents inherit permissions of users
  • Purview labels control data access
  • Granular control down to specific messages
  • Audit trails for all AI interactions
Best Practice: Ensure all sensitive data is properly labeled in Purview before enabling AI features. Test agent behavior with different sensitivity levels.
